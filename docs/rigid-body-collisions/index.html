Source snapshot

The local route vendors the published Nuxt HTML shell, the full route-local _nuxt/ chunk tree, the Essays1743 essay fonts, and the article runtime, then patches only the subpath-hosting shell and provenance surfaces needed for the public replica.

  • Live original verified against https://www.sassnow.ski/rigid-body-collisions/1 on 2026-03-21.
  • The local copy removes the body-level RSS, Twitter, e-mail, and Fathom provenance surfaces from the public body.
  • The opening collision scenes, later vector examples, and article-long interactive runtime remain driven by the vendored published assets.

Asset handoff

  • rigid-body-collisions/index.html owns the local metadata shell, shared footer hook, font rewrites, and the final route-cleanup hook for the local root.
  • rigid-body-collisions/_nuxt/ contains the full vendored Nuxt chunk graph, route CSS, and runtime dependencies required by the article.
  • rigid-body-collisions/fonts/ keeps the Essays1743 essay fonts local so the typography and route CSS no longer depend on the upstream root.
  • rigid-body-collisions/_nuxt/D4VqJVMa.js retains the published app runtime while localizing lazy asset references and disabling the dormant analytics CDN handoff.

Known deviations

  • The public route now uses local canonical and og:url metadata for subpath hosting.
  • A tiny local cleanup hook removes the vendored document-driven not-found card that appears when the published Nuxt route is flattened to the top-level local slug.
  • The shared footer is the only public original-page reference surface.

Validation guidance

  • Keep the published article structure intact; future edits should patch the shell and vendored chunks rather than rewriting the article into a new runtime family.
  • Extend the shared smoke bundle instead of creating a one-off harness so later Nuxt-backed replicas can reuse the same route checks.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching shell cleanup, font paths, or the vendored chunk tree.
