Source snapshot

The local route vendors the published article HTML, route CSS, shared article CSS, page scripts, and article images, then patches only the static-hosting shell and provenance surfaces needed for subpath hosting.

  • Live original verified against https://ciechanow.ski/alpha-compositing/ on 2026-03-20.
  • The local copy removes the upstream banner, blog navigation, analytics, Patreon prompt, and social links from the public body.
  • The interactive runtime still relies on the published canvas, slider, drag, and segmented-control modules from the vendored scripts.

Asset handoff

  • alpha-compositing/index.html owns the local metadata shell, shared footer hook, and relative article asset paths.
  • alpha-compositing/css/base.css and alpha-compositing/css/alpha_compositing.css preserve the published article layout and interactive scene styling with subpath-safe image URLs.
  • alpha-compositing/js/base.js and alpha-compositing/js/alpha_compositing.js keep the original draggers, sliders, canvas draws, and Porter-Duff scene logic.
  • alpha-compositing/images/ contains the article-local diagrams, textures, and button sprites required by the published page and scripts.

Known deviations

  • The public route no longer includes the author banner, Patreon prompt, X/Twitter, Instagram, e-mail, RSS, or Google Analytics surfaces.
  • The local copy uses local fallback font stacks instead of runtime Google Fonts requests so the route stays self-contained.
  • The shared footer is the only public original-page reference surface on the shipped route.

Validation guidance

  • Keep the article structure intact; future edits should patch the published shell, not rewrite the interactions into a custom condensed page.
  • Extend the shared smoke bundle instead of creating a one-off validator so later Ciechanowski routes can reuse the same family path.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching article shell copy or any of the vendored scripts.
