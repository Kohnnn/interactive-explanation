Source snapshot

This route is intentionally a local curation page rather than a direct mirror of one published source. It exists to unify the shipped music replicas after four families are live.

  • The hub groups Teoria, Ableton, Chrome Music Lab, and Musicmap into one browsing surface.
  • The public footer uses a neutral curated provenance label instead of pretending the hub has one original page.
  • The current editorial mix spans 15 local routes: 6 ear-training drills, 8 sequencing or composition routes, and 1 genre-context map.
  • Every body link points to a local route or a local docs page only.
  • The shared footer remains the only public provenance surface.

Asset handoff

  • music-interactive-hub/index.html owns the editorial shell, cluster copy, and recommended path.
  • The route reuses shared/site.css and shared/public-footer.js rather than introducing a new runtime family.
  • The sequencing cluster currently folds together the Ableton Playground, six Ableton lesson pages, and Chrome Music Lab Song Maker.
  • The linked experiences continue to live in their own shipped route trees and docs routes.

Known deviations

  • This hub is a local synthesis page, not a direct published-page mirror.
  • The clustering model is editorial: ear training and notation, sequencing and composition, then genre history and context.
  • The route uses local canonical and og:url metadata for subpath hosting.

Validation guidance

  • Keep all cards pointed at local routes and local docs only.
  • Update the cluster counts whenever new music routes are added or reorganized.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after changing the hub shell or route mix.
