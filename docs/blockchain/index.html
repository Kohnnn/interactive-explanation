Source snapshot

The local route vendors the interactive HTML pages from the legacy blockchain demo rather than the upstream landing page, then patches only the shell metadata, local navigation, and public provenance surfaces needed for nested static hosting.

  • Live original verified against https://andersbrownworth.com/Blockchain/blockchain.html on 2026-03-18.
  • The local root intentionally maps to the main blockchain scene, while sibling pages like hash.html, block.html, distributed.html, tokens.html, and coinbase.html remain alongside it.
  • The public route keeps the original Bootstrap and jQuery runtime intact and limits original-page provenance to the shared footer.

Anders adaptation checklist

  • Use the interactive blockchain page as the local route root instead of the upstream video landing page.
  • Vendor the demo-specific javascripts/ and stylesheets/ trees directly under blockchain/ without deduping legacy libraries.
  • Rewrite navbar links, CSS paths, JS paths, canonical metadata, and footer config so the route works from the GitHub Pages subpath.
  • Keep distributed, token, and coinbase variants available as local sibling pages under the same route tree.

Asset and source map

  • blockchain/index.html is the local blockchain root and mirrors the published blockchain.html scene with local footer metadata.
  • blockchain/hash.html and blockchain/block.html preserve the earlier hash and single-block warmup pages.
  • blockchain/distributed.html, blockchain/tokens.html, and blockchain/coinbase.html keep the broader chain variants and comparison scenes.
  • blockchain/javascripts/blockchain.js keeps the published hash recomputation, mining loop, and downstream invalidation logic.

Known deviations

  • The local route removes the upstream video landing, GitHub ribbon, author byline, and donation callouts from the public entry surface.
  • Canonical metadata and navbar links now point at local equivalents under /interactive-explanation/blockchain/.
  • The shared footer is the only public original-page reference surface on shipped pages.
