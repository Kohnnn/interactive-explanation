Source snapshot

The local route vendors the interactive key and signing pages directly rather than the upstream video landing. The local root maps to the keys page, while signatures, transaction, and blockchain variants remain available as nested local routes.

  • Live original verified against https://andersbrownworth.com/blockchain/public-private-keys/keys on 2026-03-18.
  • The local root intentionally flattens the upstream nested demo to the top-level slug public-private-keys.
  • The public route keeps the original Bootstrap, jQuery, cookie, BigInteger, Buffer, elliptic, and SHA256 runtime intact.

Anders adaptation checklist

  • Use the interactive keys page as the local route root instead of the upstream video landing page.
  • Vendor the demo-specific javascripts/ and stylesheets/ trees directly under public-private-keys/ without deduping libraries on the first pass.
  • Patch only asset paths, navbar links, canonical metadata, and footer configuration for nested hosting.
  • Keep signatures, transaction, and blockchain pages available under the flattened local slug so the original demo flow still works.

Asset and source map

  • public-private-keys/index.html is the flattened local root and mirrors the upstream keys scene.
  • public-private-keys/signatures/index.html keeps the sign and verify workflow, including the verification-state card styling.
  • public-private-keys/transaction/index.html and public-private-keys/blockchain/index.html preserve the richer transaction and signed-blockchain flows.
  • public-private-keys/javascripts/blockchain.js keeps the published cookie-backed key loading, signing, verification, and transaction logic.

Known deviations

  • The public entry route no longer surfaces the upstream video landing, author byline, or repo ribbon.
  • The local root is intentionally flattened to /interactive-explanation/public-private-keys/ even though the original demo lived under /blockchain/public-private-keys/.
  • The shared footer is the only public original-page reference surface on shipped pages.
