Source snapshot

The local route vendors the published HTML shell, hashed JS and CSS bundles, fonts, and printable cheat sheet, then patches only the subpath-hosting shell and public provenance surfaces needed for the replica.

  • Live original verified against https://qr.blinry.org/ on 2026-03-21.
  • The local copy keeps the published anatomy, mask, reading-order, bytes, length, content, and congratulations sections intact.
  • The top workshop credit line and bottom Codeberg bug or feature prompt are removed so the shared footer remains the only provenance surface.

Asset handoff

  • reading-qr-codes-without-a-computer/index.html owns the local metadata shell, shared footer hook, and route-relative hashed asset paths.
  • reading-qr-codes-without-a-computer/assets/ preserves the published hashed bundle output without reauthoring the tutorial into another shell.
  • reading-qr-codes-without-a-computer/fonts/ contains the vendored Bungee, Roboto, and Iosevka files expected by the published CSS.
  • reading-qr-codes-without-a-computer/qr-cheat-sheet.pdf preserves the published printable helper linked from the walkthrough.

Known deviations

  • The route now uses local canonical and og:url metadata for subpath hosting.
  • The shared footer is the only original-page reference surface on the public route.
  • The top workshop credit and the bottom source-code request block are pruned after the app mounts, but the decode walkthrough itself remains unchanged.

Validation guidance

  • Keep the hashed bundle intact; future edits should patch the shell or vendored assets rather than rewriting the explainer into another runtime family.
  • Extend the shared smoke bundle instead of adding a one-off harness so later non-Ciechanowski app replicas can reuse the same route checks.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching bundle paths, shell cleanup, or runtime asset files.
