Source snapshot

The local route vendors the published article HTML, route-local CSS, GSAP, Pixi, and allocator scripts, then patches only the subpath-hosting shell and provenance surfaces needed for the public replica.

  • Live original verified against https://samwho.dev/memory-allocation/ on 2026-03-21.
  • The local copy removes the upstream logo link, newsletter footer, social links, and other author-promo surfaces from the public body.
  • The opening memory scenes, allocator-backed timeline demos, and hexadecimal slider remain driven by the vendored published runtime.

Asset handoff

  • memory-allocation/index.html owns the local metadata shell, shared footer hook, and route-relative asset paths.
  • memory-allocation/css/reset.css and memory-allocation/css/main.css preserve the published article layout and component styling.
  • memory-allocation/js/memory-allocation.js, memory-allocation/js/pixi-legacy.mjs, and memory-allocation/js/gsap/ keep the published simulation runtime intact.
  • memory-allocation/js/allocators/ contains the route-local allocator implementations fetched by the article demos at runtime.

Known deviations

  • The public route now uses local canonical and og:url metadata for subpath hosting.
  • The playground callout is neutralized into plain text so the public body does not ship a non-local upstream route link.
  • The shared footer is the only public original-page reference surface.

Validation guidance

  • Keep the published article structure intact; future edits should patch the shell and vendored assets rather than rewriting the simulations into a different runtime family.
  • Extend the shared smoke bundle instead of creating a one-off harness so later Sam Rose replicas can reuse the same route checks.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching shell cleanup, allocator paths, or runtime asset files.
