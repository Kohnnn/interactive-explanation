Source snapshot

Vendor baseline came from the published ncase/sim static tree. The local copy keeps the upstream editor runtime, preset models, compressed share-link flow, and JSON export behavior while removing static creator, repo, and Patreon copy from the public shell.

  • Live original verified against https://ncase.me/sim/ on 2026-03-10.
  • The shipped route keeps the original model editor layout and script structure instead of re-skinning the tool.
  • Visible example text inside shipped model JSON files was localized so the preset descriptions no longer point at upstream URLs.

Asset and source map

  • sim/index.html provides the local shell, metadata, editor mount, and shared footer hook.
  • sim/scripts/engine/ keeps the original control bar, editor, grid, save, load, and UI runtime.
  • sim/models/ preserves the upstream preset JSONs, with local link cleanup in the visible description text.
  • sim/styles/ and the emoji helper scripts remain vendored so the tool behaves under the same relative layout as upstream.

Known deviations

  • The static metadata and visible editor copy no longer reference creator handles, TinyURL, GitHub promo copy, or Patreon language.
  • The preset example text in forest.json now uses local ./?s=... links.
  • The Schelling preset now points readers to the shipped local ../polygons/ route instead of the upstream site.

Reproduction notes

  • Vendor the upstream tree first, then patch the shell metadata, editor copy, and visible preset descriptions without rewriting the engine modules.
  • Keep the original relative file layout because the load and save helpers assume the shipped models/, scripts/, and styles/ paths.
  • Extend the shared smoke bundle instead of creating a one-off runner so future editor changes still exercise the same validation path.
