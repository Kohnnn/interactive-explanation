Source snapshot

Vendor baseline came from the published ncase/wbwwb bundle tree. The local copy keeps the original PIXI scene stack, peep behaviors, TV chyron logic, and camera capture loop while neutralizing the public credits and post-credits promo surfaces.

  • Live original verified against https://ncase.itch.io/wbwwb on 2026-03-10.
  • The shipped route uses a single explicit English runtime text map instead of the full translation bundle.
  • Public creator, supporter, and external-share surfaces were removed from the ending flow and replaced with a replay-only closeout.

Asset and source map

  • wbwwb/index.html provides the local shell, metadata, WebGL stage mount, and shared footer hook.
  • wbwwb/js/core/ and wbwwb/js/game/ keep the original scene manager, world, camera, and TV runtime.
  • wbwwb/js/scenes/ preserves the upstream preload, quote, escalation, peace, and replay structure with local cleanup in the credits scenes.
  • wbwwb/sprites/ and wbwwb/sounds/ remain vendored so the visual and audio pacing stays close to upstream.

Known deviations

  • The route ships one English text bundle instead of the original multi-language translation file.
  • The creator, playtester, Patreon, and external social-share scenes were replaced with neutral thank-you and replay cards.
  • window.SHARE_URL now resolves from the local route instead of relying on an empty or upstream-specific share target.

Reproduction notes

  • Vendor the published bundle first, then localize the shell and scene cleanup without rewriting the underlying camera or crowd systems.
  • Keep the GitHub Pages-safe relative asset layout intact; the PIXI runtime expects the original folder structure.
  • Run node interactive-explanation/tools/check-public-surface.mjs after touching text bundles or ending scenes because the audit scans those files directly.
