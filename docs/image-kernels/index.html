Source snapshot

The local route vendors the published Image Kernels page HTML, stylesheet, script, and bundled image assets, then patches pathing and public-shell markup so the original Angular runtime works under the /interactive-explanation/ subpath.

  • Live original verified against https://setosa.io/ev/image-kernels/ on 2026-03-14.
  • The local copy removes EV branding, creator bylines, share widgets, newsletter prompts, analytics, and comments.
  • The public surface still includes the upload and live-video playground because the published page ships those controls.

Shared EV layer

  • ../ev/scripts/d3.js, angular.js, and common.js are reused as the main runtime layer.
  • ../ev/scripts/exif.js is vendored locally so uploaded-image orientation handling does not depend on a remote runtime.
  • image-kernels/style.css adds only narrow-screen overflow and control-layout helpers.

Asset and source map

  • image-kernels/index.html owns local metadata, the cleaned public shell, and footer wiring.
  • image-kernels/script.js keeps the published controllers and directives for the intro matrix, inspector, and playground scenes.
  • image-kernels/resources/lxe.jpg and resources/library.jpg are vendored locally for the intro and playground defaults.
  • image-kernels/fb-thumb.png is vendored for local metadata and preview safety.

Known deviations

  • The public route no longer shows EV branding, creator links, share widgets, newsletter forms, analytics, or comments.
  • The vendored script now loads its default images from ./resources/ so the page works under the GitHub Pages-style subpath.
  • The route keeps the original upload and live-video copy instead of narrowing the surface, because that is part of the published page.
