Source snapshot

The local route vendors the published article HTML, the route-local Bootstrap sheet, the compiled D3 and React bundle, and the magnetic hysteresis image, then patches only the subpath-hosting shell and provenance surfaces needed for the public replica.

  • Live original verified against https://joshuahhh.com/projects/hysteresis/ on 2026-03-21.
  • The local copy removes the Plausible analytics script, the Cloudflare beacon, and the author link from the public body.
  • The slider scenes and the later trajectory timeline remain driven by the vendored published bundle.

Asset handoff

  • hysteresis-slack/index.html owns the local metadata shell, shared footer hook, and route-relative asset paths.
  • hysteresis-slack/bootstrap.css preserves the published article layout.
  • hysteresis-slack/build/bundle.js keeps the original D3 and React slider runtime intact while disabling the vendored dev websocket bootstrap for local static hosting.
  • hysteresis-slack/images/hyloop.gif keeps the article’s final magnetic hysteresis reference image local.

Known deviations

  • The public route now uses local canonical and og:url metadata for subpath hosting.
  • The vendored webpack dev-server socket bootstrap is disabled so the public route stays runtime-clean under static hosting.
  • The shared footer is the only public original-page reference surface.

Validation guidance

  • Keep the published article structure intact; future edits should patch the shell and vendored assets rather than rewriting the explanation into a new runtime family.
  • Extend the shared smoke bundle instead of creating a one-off harness so later small single-page replicas can reuse the same route checks.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching shell cleanup, bundle paths, or the timeline runtime.
