The local route vendors the published static linear-regression/ build from the MLU family, then patches shell metadata, local paths, and public-facing branding so the compiled runtime works cleanly under the nested replica site.

  • Live original verified against https://mlu-explain.github.io/linear-regression/ on 2026-03-14.
  • The local copy keeps the compiled Svelte runtime, KaTeX math blocks, scrollytelling observer, and chart IDs intact.
  • Creator names, MLU branding, analytics, and promo links were removed from the public route and kept in docs only.

MLU adaptation checklist

  • Keep the compiled build/bundle.js and build/bundle.css in place so the original app still mounts into document.body.
  • Patch only the string-level shell surfaces that leak upstream branding, bylines, or promo links into the public route.
  • Reuse the existing vendored MLU font payload from decision-tree/ through relative font-face paths instead of introducing a new shared family layer.
  • Let the shared footer provide the only live original-page reference link.

Asset and source map

  • linear-regression/index.html owns the local metadata shell, footer hook, and GitHub Pages-safe relative asset paths.
  • linear-regression/build/bundle.js keeps the published chart logic, scroll-driven scene changes, residual overlays, loss views, and coefficient controls.
  • linear-regression/build/bundle.css preserves the route layout, sticky chart regions, and component styling.
  • linear-regression/assets/styles/font.css now points at the already-vendored MLU and KaTeX fonts shipped with decision-tree/.

Known deviations

  • The public route no longer shows MLU branding, author bylines, analytics, or upstream promo links outside the shared footer.
  • The opening logo and closing link cluster were localized to the replica site while leaving the teaching scenes and charts unchanged.
  • The public surface keeps educational reference links that are part of the lesson, but provenance and source mapping stay in docs only.
