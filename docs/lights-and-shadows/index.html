Source snapshot

The local route vendors the published article HTML, shared article CSS, page-local CSS, scripts, and helper textures, then patches only the subpath-hosting shell and provenance surfaces needed for the public replica.

  • Live original verified against https://ciechanow.ski/lights-and-shadows/ on 2026-03-21.
  • The local copy removes the upstream banner, blog navigation, analytics, Patreon prompt, and social links from the public body.
  • The lighting, radiance, color, bounce, and occlusion scenes remain driven by the vendored scripts rather than a local rewrite.

Asset handoff

  • lights-and-shadows/index.html owns the local metadata shell, shared footer hook, and relative article asset paths.
  • lights-and-shadows/css/base.css and lights-and-shadows/css/light.css preserve the published article layout and interaction styling with subpath-safe helper image URLs.
  • lights-and-shadows/js/base.js and lights-and-shadows/js/light.js keep the original slider, drag, and lighting-scene runtime.
  • lights-and-shadows/images/ contains the shared anchor and control sprites plus the local light_shadow textures needed by the published WebGL scenes.

Known deviations

  • The public route no longer includes the author banner, Patreon prompt, X/Twitter, Instagram, e-mail, RSS, or Google Analytics surfaces.
  • The local copy uses local fallback font stacks instead of runtime Google Fonts requests so the route stays self-contained.
  • The shared footer is the only public original-page reference surface on the shipped route.

Validation guidance

  • Keep the article structure intact; future edits should patch the published shell, not rewrite the interactions into a condensed local experience.
  • Extend the shared smoke bundle instead of creating a one-off validator so later Ciechanowski routes can reuse the same family path.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching article shell copy or any vendored runtime files.
