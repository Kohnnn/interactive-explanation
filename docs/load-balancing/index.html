Source snapshot

The local route vendors the published article HTML, route-local CSS, Pixi runtime, Plotly bundle, and page script, then patches only the subpath-hosting shell and provenance surfaces needed for the public replica.

  • Live original verified against https://samwho.dev/load-balancing/ on 2026-03-21.
  • The local copy removes the upstream logo link, newsletter footer, social links, and other author-promo surfaces from the public body.
  • The opening simulations, latency graphs, and final playground remain driven by the vendored published runtime.

Asset handoff

  • load-balancing/index.html owns the local metadata shell, shared footer hook, and route-relative asset paths.
  • load-balancing/css/reset.css and load-balancing/css/main.css preserve the published article layout and simulation styling.
  • load-balancing/js/load-balancers.js, load-balancing/js/pixi.mjs, and load-balancing/js/plotly.js keep the original simulation and chart runtime intact.
  • The final playground still builds its controls dynamically from the published script rather than a local replacement panel.

Known deviations

  • The public route now uses local canonical and og:url metadata for subpath hosting.
  • The self-promotional social link in the article’s discussion note is pruned, while third-party discussion and reference links remain intact.
  • The shared footer is the only public original-page reference surface.

Validation guidance

  • Keep the published article structure intact; future edits should patch the shell and vendored assets rather than rewriting the simulations into a different runtime family.
  • Extend the shared smoke bundle instead of creating a one-off harness so later Sam Rose replicas can reuse the same route checks.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching shell cleanup, chart mounts, or runtime asset files.
