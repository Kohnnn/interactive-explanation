Source snapshot

The local route is a curated hub page built from routes that are already shipped locally. It gives the collection a focused local entry point without changing any underlying game or article runtime.

  • Live original verified against https://github.com/ncase?tab=repositories on 2026-03-21.
  • The public body links only to local replica routes and docs surfaces.
  • The shared footer remains the only public original-page reference surface.

Asset handoff

  • primary-interactive-hub/index.html owns the local metadata shell, route grouping, and curated copy.
  • The route reuses shared/site.css and shared/public-footer.js rather than introducing a new runtime family.
  • The linked experiences continue to live in their own shipped route trees and docs pages.

Known deviations

  • This route is a local hub, not a direct mirror of one published page.
  • The grouping model is editorial and optimized for local browsing, not for upstream repository ordering.
  • The route uses local canonical and og:url metadata for subpath hosting.

Validation guidance

  • Keep all body links local so the route remains a true hub for shipped replicas.
  • Update the route whenever a new qualifying local interactive is added to this family.
  • Run both node interactive-explanation/tools/check-public-surface.mjs and node interactive-explanation/tools/smoke-bundle.mjs after touching route grouping, footer handoff, or metadata.
